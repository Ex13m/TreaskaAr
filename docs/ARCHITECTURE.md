# Архитектура

## Ограничения и почему так

ТЗ челленджа фиксирует: **один файл `index.html`, vanilla JS, без сборщика**,
Three.js с CDN через importmap. Поэтому архитектура — не папки и классы,
а дисциплина внутри одного модуля: файл разбит на изолированные системы,
которые общаются через небольшое число явных точек (объект `state`,
вызовы `SFX.*`, пулы эффектов). Это сознательный выбор data-oriented
подхода (плоские объекты + системы-функции, как в ECS-движках) вместо
ООП-иерархий: меньше аллокаций, проще профилировать.

## Карта систем (порядок в файле = порядок зависимостей)

| Система | Что делает | Ключевые сущности |
|---|---|---|
| Сохранения | localStorage, semver-ключ | `save`, `persist()` |
| Рендер | WebGL, свет, ресайз | `renderer`, `scene`, `camera` |
| Фолбэк-небо | шейдерная аврора + планктон | `sky`, `plankton` |
| Обзор | гироскоп→кватернион / драг | `gyroQuat()`, `look`, `updateCamera()` |
| Звук | процедурный синтез, реверб, панорама | `AU`, `tone()`, `noiseHit()`, `spatialDest()`, `SFX` |
| Музыка | lookahead-секвенсор от комбо | `musicTick()` |
| Рыбы | инстансинг, запекание, поведение | `fishSys`, `fishes`, `updateFish()` |
| Эффекты | пулы частиц/колец/вокселей | `burst()`, `shockRing()`, `voxelBlast()` |
| Орудия | вью-модели, баллистика, отдача | `W`, `shootTurret()`, `railRelease()`, `shootShotgun()` |
| Попадания | очки, комбо, фидбек ×4 канала | `hitFish()` |
| Игровой цикл | раунд, волны, прогрессия | `state`, `updateGame()`, `startRound()`, `endRound()` |
| HUD/радар | DOM-оверлей, стрелки за кадр | `updateRadar()`, `popNumber()` |
| Ввод | tap=выстрел / drag=обзор (порог 12px) | pointer-обработчики |
| Качество | динамическое разрешение | `quality`, `qualityTick()` |

## Поток кадра

```
rAF → dt (кап 50 мс) → qualityTick
  → updateCamera (гироскоп/драг + кик)
  → updateFish (steering/boids → матрицы инстансов + фазы хвостов)
  → updateProjectiles (полёт, коллизии сферами, испуг рыб)
  → updateFX (частицы/кольца/воксели из пулов)
  → updateWeapons (пружины отката, заряд, бары чипов)
  → updateGame (таймер, спавн, комбо-окно, HUD, радар)
  → musicTick (lookahead-планирование нот)
  → renderer.render
```

Слоумо (`timeScale`) умножает dt для мира, но не для пружин оружия и
тряски — поэтому Golden Moment ощущается как «время замерло», а не «игра лагает».

## Рыбы: данные и рендер разделены

Логика держит **плоские объекты** `{pos, vel, target, anchor, wag, idx…}`
в массиве `fishes`. Рендер — пять `InstancedMesh` (по одному на вид) в
`fishSys`; связь — только целочисленный `idx` слота. Смерть рыбы — это
нулевая матрица в слоте и возврат `idx` во free-list: ни одной аллокации.

Геометрия вида запекается на старте функцией `bakeParts()`: примитивы
(сферы/конусы/цилиндры) трансформируются, получают цвет по вершинам и
склеиваются в один не-индексированный буфер. Анимация хвоста — в
вершинном шейдере (см. `fishMaterial()`): один и тот же GLSL у всех
видов (один скомпилированный шейдер), амплитуда и длина — юниформы.

## Звук: граф

```
источник (osc/noise) → gain-огибающая ─┬→ master → лимитер → выход
                                       └→ [StereoPanner] ─┬→ master
                                                          └→ send → Convolver(реверб) → wet → master
```

Все «позиционные» звуки идут через `spatialDest(pos)`: панорама из
x-координаты в камере, затухание из дистанции, посыл в реверб.
Одноразовые узлы не пулятся — их собирает GC после `stop()` (это
рекомендованный паттерн WebAudio).

## Версионирование

`VERSION` в `index.html` ↔ `CACHE = 'treska-vN'` в `sw.js` меняются вместе.
SW при активации удаляет старые кэши — у установленной PWA обновление
подхватывается на втором открытии.
